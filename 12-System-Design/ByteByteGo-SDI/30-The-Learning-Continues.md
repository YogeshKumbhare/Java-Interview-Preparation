## System Design patterns to master

| Pattern | Description |
|---------|-------------|
| Consistent Hashing | Distribute data evenly across servers |
| Rate Limiting | Protect services from abuse |
| Circuit Breaker | Prevent cascade failures |
| Service Discovery | Locate services dynamically |
| Heartbeat | Detect failed nodes |
| CAP Theorem | Consistency, Availability, Partition Tolerance tradeoffs |
| Event Sourcing | Store state changes as sequence of events |
| CQRS | Separate read and write models |
| Bloom Filter | Probabilistic membership test |
| Gossip Protocol | Peer-to-peer communication |
| Vector Clocks | Track causality in distributed systems |
| Merkle Trees | Detect inconsistencies efficiently |
| Leader Election | Coordinate distributed processes |

## Recommended books

1. **"Designing Data-Intensive Applications"** by Martin Kleppmann — The definitive guide to distributed systems concepts.
2. **"System Design Interview – An Insider's Guide"** by Alex Xu — The book this course is based on.
3. **"System Design Interview Vol. 2"** by Alex Xu — Advanced system design problems.
4. **"Building Microservices"** by Sam Newman — Complete guide to microservices architecture.
5. **"Web Scalability for Startup Engineers"** by Artur Ejsmont — Practical scalability patterns.

---

## Key takeaways from this course

1. **Understand the problem first** — Don't jump to solutions. Ask clarifying questions.
2. **High-level design matters** — Start with a high-level overview, then drill into details.
3. **There's no single right answer** — Tradeoffs are the name of the game in system design.
4. **Communication is key** — Think out loud. Collaborate with the interviewer.
5. **Practice, practice, practice** — The more systems you design, the better you get.
